수정된 파일 업데이트 절차 (3단계 명령어)
프로젝트 폴더에서 터미널(VSCode의 경우 Ctrl + ``)을 열고, 아래 명령어 3개를 순서대로 입력하시면 모든 과정이 끝납니다.
1. 변경된 모든 파일을 스테이징(Staging) 영역에 추가합니다.
이 명령어는 "이제부터 내가 수정한 이 파일들을 기록(커밋)할 준비를 하겠다"는 의미입니다.
code
Bash
git add .
(점 .은 현재 폴더의 모든 변경사항을 의미합니다.)
2. 변경 내용을 하나의 버전(커밋)으로 기록합니다.
어떤 내용을 수정했는지 나중에 알아보기 쉽도록 의미 있는 메시지를 작성하는 것이 중요합니다.
code
Bash
git commit -m "feat: 교사 모드에 전체보기 버튼 추가 및 보조 모드에 실행 취소 버튼 추가"
팁: -m 뒤의 메시지는 자유롭게 작성할 수 있습니다. 예를 들어 "버튼 위치 수정" 과 같이 간단하게 적어도 괜찮습니다.
3. 기록한 버전을 GitHub 리포지토리로 업로드(푸시)합니다.
이 명령어를 실행하면 로컬 컴퓨터에 저장된 변경 내역이 GitHub 서버로 전송됩니다. 바로 이 순간 Netlify가 변경을 감지하고 자동 배포를 시작합니다.
code
Bash
git push
그 다음은 자동입니다!
git push 명령어가 성공적으로 완료되었다면, 이제 회원님이 할 일은 끝났습니다.
Netlify 대시보드 확인: Netlify에 로그인하여 해당 사이트의 Deploys 탭으로 이동해보세요. "Building" 이라는 노란색 상태 표시와 함께 새로운 배포가 진행 중인 것을 볼 수 있습니다.
배포 완료: 1~2분 정도 지나면 배포 상태가 "Published" 라는 초록색 표시로 바뀝니다.
변경 사항 확인: 이제 회원님의 Netlify 웹사이트 주소로 다시 접속하여 새로고침하면 방금 수정한 내용이 완벽하게 적용된 것을 확인하실 수 있습니다.
결론적으로, 앞으로는 코드를 수정할 때마다 위 3단계(git add, git commit, git push)만 반복하시면 됩니다.
